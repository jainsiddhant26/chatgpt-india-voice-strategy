## 🧭 What Is This Project?

A full end-to-end PM case study exploring **why Indian students don't use voice input on ChatGPT** — and designing a solution to fix it.

The project follows a real PM workflow:
1. **Market Strategy** → Understand the opportunity
2. **User Research (Survey)** → Validate the problem with real data
3. **Problem Framing** → Synthesize insights into a clear problem statement
4. **PRD** → Design and spec the solution

---

## 🔑 The Core Insight

> *"Users are not rejecting voice — they paused because they can't verify before sending."*

| Stat | Finding |
|------|---------|
| 28% | Regular voice users |
| 48% | Tried voice but paused |
| 74% | Of all objections = "can't see/edit before sending" + accuracy concerns |
| 78.4% | Would try voice if they could see & edit transcription first |

**The problem isn't tech. It's trust and control.**

## 🎯 The Solution: Review-Before-Send

| Solution Direction | Impact | Effort | Adoption Lift |
|---|---|---|---|
| Gamification & Habit Nudges | Medium | Low | 28% → ~40% |
| Accent-Adaptive ASR | Medium-High | Very High | 28% → ~50% |
| **Review-Before-Send ✅** | **High** | **Medium** | **28% → ~78%** |

**Chosen solution:** Show a fully editable transcription box before the message is sent — giving users control and confidence in high-stakes moments.
